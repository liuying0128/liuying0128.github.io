什么是npm

npm (node package management)

npm 为你和你的团队打开了连接整个 JavaScript 天才世界的一扇大门。它是世界上最大的软件注册表，每星期大约有 30 亿次的下载量，包含超过 600000 个 包（package） （即，代码模块）。来自各大洲的开源软件开发者使用 npm 互相分享和借鉴。包的结构使您能够轻松跟踪依赖项和版本。

下面是关于 npm 的快速介绍：

npm 由三个独立的部分组成：

  • 网站
  • 注册表（registry）
  • 命令行工具 (CLI)

[网站](https://npmjs.com/) 是开发者查找包（package）、设置参数以及管理 npm 使用体验的主要途径。

注册表 是一个巨大的数据库，保存了每个包（package）的信息。

[CLI](https://docs.npmjs.com/cli/npm) 通过命令行或终端运行。开发者通过 CLI 与 npm 打交道。

用npm可以…

  • 将包应用到你的应用中，或者组织合并它们
  • 下载可独立运行的工具，并且可以立即使用它们
  • 不下载就可以使用npx命令运行包
  • 在任何地方和其他npm使用者分享代码
  • 对特定的开发者限制代码
  • 组织虚拟团队
  • 管理各种代码版本和代码的依赖

安装npm

npm is written in Node.js, so you need to install Node.js in order to use npm.
NPM是用Node.js编写的，因此您需要安装Node.js才能使用NPM。

You can install npm via the Node.js website, or by installing a Node Version Manager or NVM. This chapter explains both options.
您可以通过Node.js网站或通过安装Node版本来安装NPM或NVM。本章解释了这两种选择。

If you just want to get started exploring npm, using the Node.js installation method is fastest. If you are an advanced developer ready to jump in and work with versions, use the node version manager. If you aren’t sure, please read this chapter before you decide. You can always change how you run npm in the future.

如果您只想开始研究NPM，使用Node.js安装方法是最快的。如果您是高级开发人员，准备在版本上投入，就使用NVM。如果你不确定，请在你决定之前先读一下这一章。在未来你总是可以改变自己怎么使用npm的。

使用cmd命令进行配置
启动cmd，输入

npm config set prefix "D:\ComputerScienceAndTechnology\Environment\NodeJS\nodejs\node_global"

npm config set cache "D:\ComputerScienceAndTechnology\Environment\NodeJS\nodejs\node_cache"

如果不进行这一步设置，npm的全局安装包，将不会在node安装文件夹里。
